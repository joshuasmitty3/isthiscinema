Movie Watch List App Requirements
Overview
A personal movie tracking web application for two users to search, save, and track movies they want to watch and have watched. The app will be built using React and hosted on Replit.
Core Functionality
Authentication

Simple shared Gmail login for both users

Search

Integration with OMDB API to search for movies by title
Display search results with movie options based on title
Ability to add selected movies to the "Watch" list

Movie Data

Each movie should display:

Artwork/poster
Synopsis
Year
Director


Movie details should be accessible from any list by clicking on a movie card

Watch List

Display movies the users plan to watch
Allow manual re-ordering of movies (drag and drop)
Enable removing titles from the list
Provide option to move a movie to the "Watched" list after viewing

Watched List

Display all watched movies in the order they were watched
Include a hidden notes section for each movie that appears when selected
Allow adding a short review (movie title + review = max 140 characters)
Store the date when the movie was moved to the "Watched" list

Tracking

Record all watched movies
Track watch dates (when added to the watched list)
Store short reviews

Data Management

Use Replit Database for data persistence
Implement CSV export functionality for backup purposes

Design & UX
Visual Design

Mobile-optimized web app (responsive design)
Simple, clean interface with curved edges and card-based UI
Movie posters should fit mobile screen without scrolling
Allow zooming in on posters
Use dusty tones

Performance

Responsive interface for adding, removing, and re-ordering movies
"Snappy" feel for all interactions

Technical Constraints

Built with React
Use Replit for hosting and database
OMDB API integration via fetch calls
No offline functionality required
No notifications system needed
No rating system beyond text reviews
No additional filtering/sorting beyond manual ordering

Non-Functional Requirements

Built for personal use by two users only
Stability and consistency prioritized over scalability
No social elements or structure for growth
